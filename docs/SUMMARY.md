# zllm — Project Summary

A from-scratch Rust LLM inference engine + white-box control plane. This doc is the
fast on-ramp for a new session (human or AI): what exists, what's proven, what's a
dead end, and which pieces transfer to other LLM work. For numbers see
[BENCHMARKS.md](../BENCHMARKS.md); for history see [CHANGELOG.md](../CHANGELOG.md).

## Release status: v1.0.0-rc.1 (2026-07-13)

The [V1_PLAN](V1_PLAN.md) executed M1–M5, each exit criterion verified live:
honest OpenAI surface (stop strings / penalties / logit_bias / seed on every
lane; unsupported params 400) with a conformance suite; **multi-arch**
(llama-family + Qwen2 via the `backend/arch.rs` registry) and **single-file
GGUF loading** (embedded vocab oracle-proven token-identical, embedded chat
templates via minijinja, GGUF-declared stops); **model-true context** (16.5K
needle retrieved on the 1B; chunked prefill; linear/YaRN/llama3 RoPE scaling);
hardening (loopback default, `ZLLM_API_KEY`, 503 backpressure, wall-clock
budgets — live robustness suite incl. SSE-disconnect slot release); Docker-
validated packaging + declared API stability tiers. **Gate to 1.0.0: the RC
soak week**, plus the gpu/vulkan monolith split as background work.

## Target & the one constraint that shapes everything

- **Box:** AMD Ryzen AI MAX+ 395 "Strix Halo" — Zen 5, 16C/32T, AVX-512 (VNNI/BF16),
  ~96 GB unified LPDDR5X (**~256 GB/s, shared by CPU + iGPU**), Radeon 8060S iGPU
  (RDNA 3.5, 40 CUs, fp16 coopmat peak ~59 TFLOP/s). Windows 11 / MSVC.
- **Model:** Llama-3.2-1B-Instruct, primarily **all-Q4_K** GGUF (663 MiB).
- **The constraint:** the LPDDR5X bus. **Decode is memory-bandwidth-bound**
  (~663 MB of weights streamed per token); **prefill is compute-bound** (coopmat
  GEMM). Because CPU and iGPU share the bus, heterogeneous decode is zero-sum.
  Every result below follows from this.

## Architecture: three backends + a control plane

| backend | feature flag | role | entry point |
|---|---|---|---|
| **CPU (candle)** | default | always-available fallback; custom AVX-512 Q4_K | `src/backend/candle/` |
| **wgpu / WGSL** | `gpu` | the **serving** stack (continuous batching) | `src/backend/gpu/mod.rs` |
| **raw Vulkan / ash** | `vulkan` | the **deployed fast single-stream** path (`ZLLM_VK=1`) | `src/backend/vulkan/mod.rs` |

Shaders are WGSL compiled to SPIR-V **in-process via `naga`** (no Vulkan SDK /
glslang / cmake in this environment — this was the key unblock; committed `.spv` are
generated by `--ignored` gen tests). The control plane (hooks, FSM, scheduler,
goals) sits above the backend `trait` and works with any of them.

## Inference work — status

### Single-stream decode — SHIPPED, beats llama.cpp
- Custom Q4_K decode matvec (workgroup-per-row + shared-mem reduce; ~84% of peak BW),
  decomposed FFN (`w13`/`w2`), **barrier-lean flash-decode SDPA**, **GPU argmax**
  (4-byte readback vs 128k-wide logits, ~40% fewer bytes/token on greedy).
- **1.12× llama.cpp short-context** (209.7 vs 187.1 tok/s sustained), bit-exact vs candle.
- Long-context was 0.94–0.96× → see **head-major KV** below, which recovers it.

### Long-context decode — head-major KV cache (`ZLLM_HEADMAJOR_KV`, opt-in)
- Cache laid `[kv_head, pos, hd]` instead of `[pos, kv_head, hd]` so each attention
  workgroup reads its KV-head **contiguously** (~187 vs ~100 GB/s cold). **+8.7%
  @ctx512, +9.9% @ctx2048, +12.4% @ctx3072** (grows with depth), bit-exact.
- Deployable via both cache-fill paths (decode-fill is native; batched
  `prefill_forward` transposes once at the prefill→decode boundary, <1% of prefill).
- **Generalized to any hd≤64 model** (dims read from the uniform; one SPV).
- **Opt-in on purpose:** it forces always-flash, which *loses* at very short context
  (~12 tok: 160 vs 172 single-pass). It's a long-context (RAG/agents) optimization.

### Prefill — real 2× gain, not yet parity
- Batched coopmat (KHR_coopmat) Q4_K GEMM. ~18% of WMMA peak (llama ~25%) — a
  pipelining gap, **0.38–0.49× llama** (was 0.25×). A hard kernel fight, not a wall.

### Serving — full vLLM-style stack (`gpu` backend, `ZLLM_CB=1`)
- Slot-indirected batched decode, **paged KV**, **prefix cache with LRU reclaim**
  (9× hot-prefix hits under overcommit), **preemption**, **chunked prefill**,
  **batched spec-decode**, **hot-swap**. **5.6× aggregate @ 8 concurrent.**
- Raw batched *throughput* parity with llama is blocked: small-M (8) skinny Q4 GEMM
  is **LDS-bandwidth bound** — a hardware wall, not a code gap.

### CPU decode — parity with llama.cpp
- 50 → **63 tok/s** (llama.cpp CPU 64) via **physical-core pinning** (`core_affinity`;
  SMT oversubscription was a ~33% tax on bandwidth-bound decode). `ZLLM_NO_PIN=1` to disable.

### Quantization
- Q4_K repack (`BlockQ4Kx8`), bit-exact Q8_K quantization, AVX-512 VNNI Q4_K×Q8_K dot.
- Sub-Q4 sensitivity harness (held-out perplexity, per-tensor requant). Finding:
  gate+up→Q3 = +2.7% ppl on Q4-pure, but **the Q3 decode kernel measured parity**
  (Q3_K unpack ALU offsets the fewer bytes on bandwidth-bound decode — the
  bytes/token lever doesn't pan out). `ffn_down→Q2` is a quality cliff.

## White-box control plane (model-agnostic, above the backend trait)
*(Status is honest — several README claims were scaffolding; this reflects reality
as of the 2026-06 feature audit.)*
- **Mid-layer hooks** (`engine/hooks/`, `runner_observer.rs`) — zero-copy
  hidden-state access during prefill, with a **write-back** path (callback returns
  `Option<Tensor>`; the observer applies hook `residual_delta`s to the full residual
  stream). Real: capture, **early exit** (live), **memory inject** (wired, but
  **gated off by default** — `engine.memory_inject_alpha = 0.0`; live A/B showed
  injecting uncurated captures at 0.3 derails a 1B), **steering** (mechanism +
  tested; not yet registered into a live request path).
- **Hallucination/uncertainty detection** (`engine/hallucination.rs`) — output-
  distribution entropy/top-prob, opt-in `detect_hallucination`. Real + live-verified
  (confabulated fact flagged at 0.63 risk vs factual 0.44). Caveats: uncertainty
  proxy (hidden-state probe is the planned v2); signal blurs with warm slot-cache
  state — cold-prefill for reproducible reports.
- **Sampler** (`sampler.rs`, real) + **speculative decode** (`spec_decode.rs`,
  prompt-lookahead drafting, real).
- **Grammar-constrained decoding** (`logit_fsm.rs`) — **`regex:` is real** (anchored
  byte-DFA token masking; EOS only at a complete match; live-verified exact-shape
  outputs) plus `ban:<ids>`. JSON-schema/BNF not implemented (rejected with 400).
  Gotcha: regex-automata doesn't merge match-unreachable states into dead —
  viability needs a match-reachability fixpoint, and use `MatchKind::All`.
- **Goal/task/status API** (`control_plane/goal_manager.rs`) — REST `/v1/goal/*`,
  disk-persisted, backed by `engine/memory_store.rs`. *(REST, not gRPC; gRPC removed v0.4.)*
- **Paged-KV continuous batching** is the real serving stack — in the **GPU backend**
  (`backend/gpu`), no tenant isolation (tenancy removed v0.3).
- **Adaptive Latent Reasoning** (`runner.rs` 3-zone loop) now runs on the real
  per-layer forward: `Backend::embed_tokens` seeds the residual stream,
  `forward_layer` executes actual transformer blocks (candle fork
  `forward_one_layer`, validated bit-exact vs the fused pass), and
  `compute_logits` projects through the real final-norm + LM head. Zone
  boundaries clamp to the loaded model's depth. Whether the 3-zone loop
  *earns its keep* vs the plain chat path is still an open product question —
  but it is no longer dead code on stubs.
- **REST/OpenAI server** (`src/server/rest.rs`): `/v1/completions`, chat, CB endpoints.

## Reusable techniques (transfer to other LLM engines / sessions)
- **`naga` in-process WGSL→SPIR-V** — ship GPU kernels with no Vulkan SDK/glslang.
  Caveat: naga 22.1's SPIR-V backend **can't emit `override`/spec-constants** — pass
  variable dims via uniforms instead (or template the source + recompile at load).
- **ggml block-format unpacking in-shader** — Q4_K (144 B), Q3_K (110 B), Q6_K; incl.
  the 12 B→16×6-bit scale shuffle (`kmask1=0x03030303`, `kmask2=0x0f0f0f0f`).
- **Head-major KV layout** for contiguous per-head cold reads at long context.
- **Atomic grid-sync is coherent on RDNA3** (L2 is the coherence point) — enables
  persistent/megakernel patterns despite WGSL having no device-scope barrier.
- **vLLM-style serving primitives**, each independently reusable: paged KV, LRU
  prefix cache, swap/preempt, chunked prefill, batched spec-decode.
- **Record-once Vulkan command buffer** — re-record only when the SDPA grid /
  lm/argmax tail changes; per-token, only mapped uniforms update.
- **Physical-core pinning** for bandwidth-bound CPU kernels (SMT hurts).
- **Quant-sensitivity harness** — per-tensor requant + held-out perplexity gate.

## Dead ends / refuted (don't re-chase — all built & measured)
- **Megakernel:** foundation works (grid-sync coherent, bit-exact PoC) but
  **architecturally marginal** — persistence caps resident workgroups (~512–1024)
  while matvecs need ~16384 for full BW, so it grid-starves its own matvecs; the
  grid-sync cost (~0.27 ms/tok @512wg) also cancels the dispatch saving.
- **Heterogeneous prefill (CPU+iGPU):** refuted — they contend on the bus (1.85×).
- **Q3 gate+up kernel:** parity (dequant ALU offsets byte savings).
- **Fused-QKV** (decode + prefill): parity.
- **Swap-to-host preemption:** moot on UMA.
- **Tree drafting:** ≈0 branching gain on real output (use linear prompt-lookahead).
- **Batched-throughput parity:** skinny-GEMM LDS-bandwidth wall (hardware).

## Methodology lesson (the most transferable thing)
**Isolated microbenchmarks mislead — only in-forward measurement is reliable.**
At least 7 times this work, an isolated bench predicted a win/wall that the deployed
forward contradicted (REQUIRE_FULL_SUBGROUPS artifacts, cache-inflated decode floors,
kernel-id mismatches, an *assumed* megakernel wall that a 1-commit PoC disproved).
Measure interleaved + averaged in the real forward; treat microbenches as hypotheses.

## Key flags
`ZLLM_VK=1` (deployed Vulkan path) · `ZLLM_CB=1` (continuous batching) ·
`ZLLM_HEADMAJOR_KV=1` (long-context KV) · `ZLLM_NO_PIN=1` (disable core pinning) ·
`ZLLM_MODEL=<path>` / `ZLLM_CTX=<depth>` (test harnesses).
Reproduce the headline win: `ZLLM_VK=1 zllm generate -m <all-Q4>.gguf -t tokenizer.json -p "..."`.
